## What Was Discovered

### 1. Existing Drafts Don't Follow Protocol Format

The 7 existing episode drafts use **table format** for the Node Register instead of the protocol-specified format:

**Current format (in drafts):**
```markdown
## 4. Node Register

| ID | Entity / Target | Evidence Count | Claim Count |
|----|------------------|----------------|-------------|
| N-1 | Charlie Kirk | 0 | 1 |
| N-2 | Erica Kirk | 10 | 7 |
```

**Protocol-specified format:**
```markdown
## 4. Node Register

**N-1** Charlie Kirk

Short descriptive line explaining why this node matters.

Evidence Count: 0
Claim Count: 1
Episode Count: 1

*Related: A-1000.1, C-1009*
```

This is why:
- **0 Person nodes** and **0 InvestigationTarget nodes** were ingested
- **0 INVOLVES relationships** were created
- **93 claims** show as having "no related nodes"

### 2. Some Claims Lack Artifact Anchors

25 claims (mostly in Episode 2) don't have `ANCHORS` relationships from artifacts. This could be:
- Missing `Anchored Artifacts:` lines in the markdown
- Claims that were inscribed without proper artifact backing (protocol violation)

### 3. Minor Orphaned Artifacts

3 artifacts have no relationships:
- `A-1037.1` — Johnny's Locker Program Description
- `A-1040.1` — Everyday Heroes Partnership with Tracy Martin  
- `A-1048.3` — Proclaim LLC Corporate Filing

These may be artifacts that were shown but not used to anchor any claims.

- Some claims may not have proper artifact anchoring
