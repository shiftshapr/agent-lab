How this stays in sync with your work
Cards come from the FINDINGS list inside scripts/transcript_corrected_quality_report.py. They are not auto-generated from git diff or from “Implement” JSON you already applied. When you fix something in transcripts_corrected/*.txt (or via apply_quality_export_to_corrected.py), delete or update that finding in the script, then regenerate — otherwise the page will keep showing it.

What this is: each card pulls a live excerpt from transcripts_corrected/ (matched by anchor substring), with an editable note on the right. Open Video to hear that moment. Tick Implement (or edit text) and use Copy JSON / Download JSON for a machine-readable queue.

Tiers: Low = review first (integrity / serious issues). Medium = ads and long segments to trim. High = small typos or optional polish. Cards with a purple border = anchor text not found (transcript changed) — edit FINDINGS in transcript_corrected_quality_report.py and regenerate.
